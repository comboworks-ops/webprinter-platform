2. Runtime Flow

The publishing path is the critical line for branding, preview, and storefront behavior.

Admin UIUser edits settings, pages, colors, content, and product visibility.
Draft StateTemporary changes live here first and should be safe to test.
Preview RuntimePreview reflects draft and published combinations.
Publish ActionApproved state is pushed forward as tenant configuration.
Tenant Published ConfigPublished tenant state becomes the runtime source.
Live StorefrontCustomer sees final output and stable pricing/product behavior.

5. API Hardening Sequence

Expose stable behavior behind contracts. Do not export instability into APIs.

1. StabilizeFix module behavior in UI/runtime until it is reliable.
2. Declare ReadyMark the module ready for API hardening.
3. Define ContractRequest/response, errors, tenant scoping, rollback.
4. Read-only APIWrap existing logic first. No rewrite.
5. Shadow CompareOld path and API path must match under real data.
6. Feature FlagRoll out safely. Writes come later.
→→→→→
